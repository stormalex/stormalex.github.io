Linux USB系统分析

USB系统中软件部分分为：

  • USB主机控制器驱动
  • USB通用设备驱动
  • USB接口驱动
  • root hub驱动

    USB硬件部分分为：

  • USB主机控制器（OHCI， EHCI， UHCI）
  • USB设备，包括一般的设备和usb hub
  • USB接口
  • root hub（之所以把根hub单独列出来是因为linux中有单独的根hub的驱动）

    这里分析USB接口驱动注册过程。

    2.USB接口驱动初始化过程

    上面分析了第一条线，也就是USB设备驱动的执行流程，假设已经有一个USB设备在了系统上。 接下来分析USB接口驱动注册的过程，假设已经有一个可用的USB设备插入了系统

    USB设备驱动由一个 struct usb_driver 结构体代表

    
struct usb_driver {
    const char *name;

    int (*probe) (struct usb_interface *intf,
              const struct usb_device_id *id);

    void (*disconnect) (struct usb_interface *intf);

    int (*unlocked_ioctl) (struct usb_interface *intf, unsigned int code,
            void *buf);

    int (*suspend) (struct usb_interface *intf, pm_message_t message);
    int (*resume) (struct usb_interface *intf);
    int (*reset_resume)(struct usb_interface *intf);

    int (*pre_reset)(struct usb_interface *intf);
    int (*post_reset)(struct usb_interface *intf);

    const struct usb_device_id *id_table;

    struct usb_dynids dynids;
    struct usbdrv_wrap drvwrap;
    unsigned int no_dynamic_id:1;
    unsigned int supports_autosuspend:1;
    unsigned int soft_unbind:1;
};


    
usb_register_driver(struct usb_driver *new_driver, struct module *owner, const char *mod_name)
    //首先设置驱动对象中的drvwrap，而再看设备驱动中也有drvwrap，这个就是用来区分设备驱动和接口驱动的，具体通过drvwrap.for_devices这个变量进行标识
    new_driver->drvwrap.for_devices = 0;
    new_driver->drvwrap.driver.name = (char *) new_driver->name;
    new_driver->drvwrap.driver.bus = &usb_bus_type;             //接口驱动的bus
    new_driver->drvwrap.driver.probe = usb_probe_interface;     //接口驱动的probe函数
    new_driver->drvwrap.driver.remove = usb_unbind_interface;
    driver_register(&new_driver->drvwrap.driver);       //注册到系统中

                            drv->bus->match(dev, drv)   //这里执行usb_bus_type中的match函数
                                usb_device_match(dev, drv)  //这里不分析了，过程和USB设备和通用设备匹配一样，只是现在传入的drv是一个USB接口驱动，需要匹配USB接口
                        driver_probe_device(drv, dev);  //如果匹配到USB接口
